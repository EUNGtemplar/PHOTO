# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

A static, single-page portfolio site for 박현응 (Yukinian job application). No build step, no
framework — plain HTML/CSS, plus GSAP core + `ScrollTrigger` + `ScrollSmoother` loaded from CDN
`<script>` tags for the hero character reveal and the page-wide smooth-scroll/pin system (see
"Hero title character reveal" and "ScrollSmoother wrapper" below).

## Commands

- **Run**: open [index.html](index.html) directly in a browser, or serve it locally, e.g.
  `npx http-server .` / `python -m http.server`, then visit the printed URL.
- **Build/lint/test**: none — there is no toolchain in this repo.

## Architecture

- [index.html](index.html) — all page markup, in section order: intro brand splash → sticky nav →
  hero → career timeline → about → strengths → job-match ledger → contact → footer. Section ids
  (`#top`, `#career`, `#about`, `#strengths`, `#match`, `#contact`) are the nav
  anchor targets — note career comes first in both the DOM and the nav, ahead of about, by design
  request. The numbered `mlabel` prefixes (01–05) reflect this order; keep them in sync if sections
  are reordered again.
- [css/style.css](css/style.css) — all styling. Design tokens (colors, fonts, spacing) live as CSS
  custom properties in `:root` at the top of the file, adapted from a black-canvas /
  single-gold-accent design system. `.mlabel`, `.btn`, `.card`, `.ledger__*`, `.timeline__*` etc.
  are the reusable block classes; keep new sections consistent with these rather than introducing
  one-off inline styles. `--section-pad-y` controls the vertical breathing room
  between sections — bump it there rather than per-section if the whole page needs more/less air.
- `#top, #about, #strengths, #match, #career, #contact { scroll-margin-top: ... }` exists
  specifically so anchor-jump/scrollIntoView navigation clears the sticky/pinned nav — the
  mobile breakpoint uses a larger value because the nav wraps to two lines at narrow widths. Keep
  this in sync if the nav's height or wrap behavior changes. Only really matters for the native
  `scroll-behavior: smooth` fallback path (no GSAP) — when `ScrollSmoother` is active, nav
  clearance is instead computed in JS (see "ScrollSmoother wrapper" below).
- `.intro` is a full-viewport (`100svh`) brand splash before the nav — solid `--gray-900` bg, no
  imagery. Wrapped in `.intro-scroller` (`height: 420vh`), pinned via `ScrollTrigger.create({
  trigger: '.intro-scroller', start: 'top top', end: 'bottom bottom', pin: '.intro', pinSpacing:
  false })` — a GSAP pin, not CSS `position: sticky` (see "ScrollSmoother wrapper" below).
  `onUpdate` maps `self.progress` (0–1) to 7 stages via `setStage`: fixed "WE ARE" label + swapping
  word (Sense → Style → Synergy → Culture → Humanity → "BE UNIQUE LIKE", the last via
  `.intro__seq-word--final`), each transition using `.is-swapping` opacity/translateY; the final
  stage fades the sequence out and `.intro__logo` in. `.intro__sequence`/`.intro__logo` share one
  grid cell (`grid-area: 1 / 1` on `.intro__content`) so the crossfade causes no layout shift.
  `.intro__scroll` links straight to `#top` as a skip escape hatch. Under `prefers-reduced-motion`
  or missing `window.gsap`/`ScrollTrigger`, JS collapses `.intro-scroller` to `auto` height and
  jumps to the static "YUKINIAN®" state. `aria-hidden` — decorative only.
- [img/](img/) — image assets: the profile photo (`Hyuneung_black.png`, used in `#contact`) and
  `og-image.png` (1200×630 link-share preview card — black canvas, "bright." wordmark, generated
  via a one-off PowerShell + `System.Drawing` script, not hand-designed or regenerated by any
  build step). Referenced with plain relative `<img>`/`<meta>` tags — no build step processes or
  optimizes these.
- **Open Graph / Twitter Card tags** in `<head>` point `og:image`/`twitter:image` at
  `img/og-image.png` specifically so link previews (KakaoTalk, Slack, etc.) show that brand card
  instead of falling back to scraping the page for the first `<img>` — which used to be the
  profile photo in `#contact`. If the page's title/description copy changes, keep `og:title`/
  `og:description`/`twitter:*` in sync with the `<title>`/`<meta name="description">` above them.
- **About scattered accent photos**: `#about` also holds six `img/Yu1.png`…`Yu6.png`
  (`.about__card--1`–`--6`), absolutely positioned in `.about__cards` (`position: absolute; inset:
  0; z-index: 0`) in the section's side gutters — hand-placed `top`/`left`/`right` %, each with a
  small CSS `rotate` (not `transform`, so it doesn't fight GSAP's transform tweens), overlapping
  behind `#about .section__inner` (`z-index: 1`); `#about` is `position: relative; overflow:
  hidden` to contain them. No card chrome, `pointer-events: none`, `aria-hidden`. An IIFE
  fades/settles each card in (`opacity`/`y`/`scale`, `scrub: 1`) as the section scrolls past —
  **not pinned** like the intro, since pinning would force `#about` to a fixed height and its
  variable-length body text needs `min-height`, not fixed `height` (see `.intro`'s `100svh`
  above); each card's `ScrollTrigger` range is `(8 + i * 4) + '% center'` through `+40%` of
  `#about`'s own height, staggered 4% apart, so all six accumulate. Hidden below `900px` width. No
  base `opacity: 0`, so the `prefers-reduced-motion`/no-GSAP fallback renders the cards fully
  visible at their static position.
- **Film-grain overlay**: `.grain` is a `position: fixed`, full-viewport, `pointer-events: none` div
  sitting above everything (`z-index: 200`) with `filter: url(#grainNoise)` (an `feTurbulence` +
  desaturate filter defined in a hidden `<svg>` right after `<body>` in `index.html`) and
  `mix-blend-mode: screen` at low opacity. It's static, not animated. If it ever looks invisible
  again, check the blend mode first — `overlay`/`soft-light` barely affect near-black pixels, which
  is most of this page; `screen` is what makes grain read against the black canvas.
- **Scroll reveal**: elements with class `.reveal` start hidden/offset and get `.is-visible` (via a
  small vanilla-JS `IntersectionObserver` at the end of `index.html`) the first time they enter the
  viewport, producing a fade/rise-in as the page is scrolled. Respects
  `prefers-reduced-motion`. Applied to every `<section>`/`<footer>` after the intro except
  `#strengths` — not the hero header (it has its own gated reveal, see below), the intro itself
  (already visible on load), or `#strengths` (its cards get their own per-card scroll reveal
  instead, see "Strengths split layout" below; this has been true since the very first commit, not
  something this redesign changed).
- **Giant-title parallax**: elements with `data-parallax="<speed>"` (`.hero__title`, every
  `.section__title`) get a `translateY` offset proportional to distance from the viewport's
  vertical center (`(viewportHeight/2 - elementCenter) * speed`), via a `rAF`-throttled
  scroll/resize listener. Deliberately **not** based on raw `window.scrollY` — that grows
  unbounded with page position and pushed later titles up over the `.mlabel` eyebrow above them.
  Skipped under `prefers-reduced-motion`.
- **Nav active-state**: a third `IntersectionObserver`, built from the `.navlink` hrefs so it stays
  in sync if nav links change, toggles `.navlink--active` on whichever nav link's section is
  currently centered in the viewport (`rootMargin: '-40% 0px -55% 0px'`).
- **Hero title character reveal**: `.hero__title` ("bright.") is markup as one `<span
  class="char">` per letter. A guarded IIFE (`window.gsap`/`ScrollTrigger` +
  `prefers-reduced-motion` checks) uses `ScrollTrigger` with `scrub: 1` (a 1s catch-up lag, not
  rigid `scrub: true`) to animate each `.char` (opacity/yPercent/scaleX/scaleY) as the title
  crosses the viewport middle, reversible on scroll-up. GSAP (core + `ScrollTrigger` +
  `ScrollSmoother`, via CDN) is the only external dependency — everything else is vanilla JS. Needs
  no special handling when `ScrollSmoother` is active: `ScrollTrigger` reads its virtualized scroll
  position automatically.
- **Hero entrance parallax**: `.hero__main` (title + copy block) scales `0.5→1` and settles
  `yPercent 12→0` as the hero scrolls into view — a separate `gsap.fromTo` scrub (`scrollTrigger: {
  trigger: '.hero', start: 'top bottom', end: 'top center', scrub: 1 }`, same 1s catch-up lag as
  the char reveal). Composes with (doesn't conflict with) the per-character reveal since it
  transforms the parent while chars animate inside it. Same guard pattern — skipped under
  `prefers-reduced-motion` or missing GSAP.
- **Gated hero reveal**: the rest of the hero (`.hero__eyebrow`/`.hero__kicker`/`.hero__lead` — the
  hero is now just the title plus these, since the photo/name/intro-text moved to `#contact`, see
  below) starts `opacity: 0` and fades in once `.hero` gets `.is-revealed` — added by the
  char-reveal `ScrollTrigger`'s `onUpdate` the first time scrub progress hits ~100%. **One-shot**:
  a closured flag, never reset, so scrolling back up doesn't re-hide it. If GSAP fails to load or
  reduced-motion is on, `.is-revealed` is added immediately. The four elements slide up
  (`translateY(44px) → 0`) via plain CSS `transition-delay` (`.1s`/`.2s`/`.3s`) even though
  `.is-revealed` hits all four at once — no JS stagger needed. Not timed against the intro's logo
  fade-out (separate scroll ranges, `.nav`'s pin sits between). Anchor scrolling (`scroll-behavior:
  smooth`) is the fallback when `ScrollSmoother` isn't active; when it is, `a[href^="#"]` clicks
  are intercepted and re-issued through `smoother.scrollTo()` instead (native smooth-scroll and the
  smoother's transform-based scroll would otherwise fight).
- **Contact intro**: `#contact` opens with `.contact-intro` (photo + name + one-line pitch) right
  after the "get in touch." title, before `.contact-grid`. This reuses the `.hero__photo`/
  `.hero__intro-*` classes that used to live in the hero header — they were relocated here, not
  duplicated, so there's only one photo+bio block on the page now. Because those class names are
  no longer inside `.hero`, the "Gated hero reveal" opacity rule above intentionally does **not**
  reference `.hero__photo` or `.hero__intro-*` — it only lists `.hero__eyebrow`/`.hero__kicker`/
  `.hero__lead`. `.contact-intro`'s visibility instead comes from `#contact`'s own `.reveal`
  fade-in, same as the rest of that section.
- `.skip-link` is a visually-hidden-until-focused "본문 바로가기" link, first element in `<body>`,
  jumping keyboard/screen-reader users past the intro splash and nav straight to `#top`.
- **Strengths split layout**: `#strengths` wraps its content in `.section__inner` (max-width
  1180px, centered — the same wrapper every other section uses; without it the section runs
  full-bleed to the viewport edge on wide monitors instead of aligning with the rest of the page)
  around `.strengths-layout` (CSS grid, 1fr/1fr — even split so the gap gutter lands on the
  section's horizontal center). The left column is two nested elements, not one:
  `.strengths-aside-cell` is the actual grid item, stretched by the grid's default
  `align-items: stretch` to the same height as `.strengths-cards`; `.strengths-aside` (the
  `03 / 핵심 강점` label + title, `text-align: right` so both sit close to the cards column, reverts
  to `text-align: left` in the sub-900px collapsed layout) is nested one level inside that. That
  nesting exists so `.strengths-aside-cell` can `display: flex; align-items: flex-start` and anchor
  `.strengths-aside` to the top of the cell's full stretched height (same height as
  `.strengths-cards`) — this used to be done with `position: sticky` on `.strengths-aside` itself,
  but GSAP's own docs warn against combining `position: sticky` with `ScrollTrigger.pin` on the same
  element: sticky's scroll-dependent bounding rect confuses the pin's `'center center'` start-point
  math (verified empirically — it pinned the title against the *bottom* of the viewport instead of
  the center). Flex-alignment instead gives `.strengths-aside` a plain, scroll-independent natural
  position, which is what the active path — a `ScrollTrigger.pin` on `.strengths-aside` in main.js,
  same reason `.nav`/`.intro` use `pin` instead of relying on sticky (see "ScrollSmoother wrapper"
  below: ScrollSmoother transforms `#smooth-content` rather than moving real `scrollTop`, so native
  sticky never sees itself as scrolled) — measures from. That pin's trigger `start: 'center center'`
  (not `'top top'`) is what makes the pin hold the title vertically centered in the viewport: it
  starts pinning the moment `.strengths-aside`'s own center crosses the viewport's center as the
  section scrolls in, then holds that exact screen position until `end` — independent of where the
  natural position was, so the title still ends up perfectly centered once pinned either way. `end`'s
  `endTrigger` is card 04's own `.card__meta` label (`#strengths .card:last-child .card__meta`), not
  `.strengths-cards` itself — the column's bottom edge trails the label by that card's own bottom
  padding, so pointing at the column released the pin a few px later than "YUKINIAN / 04-04" itself
  had scrolled past. Top
  alignment (not center) is what it's anchored to, though, because the stretched cell spans all four
  cards: centering `.strengths-aside` inside that full height put its natural position — and so the
  pin's trigger point — far down the document, past where card 01 had already scrolled out of view
  by the time the pin engaged. Anchoring to the top keeps the title's natural position near card 01,
  so the pin now engages while card 01 is still on screen. The flex-alignment also
  doubles as the no-JS/`prefers-reduced-motion` fallback — without JS the title just renders at the
  top of its column instead of tracking scroll (not truly "sticky," but visually reasonable for
  a rarely-hit degrade path). That pin is wrapped in
  `ScrollTrigger.matchMedia({'(min-width: 901px)': ...})` rather than a one-time `innerWidth` check,
  so it's torn down and recreated if the viewport crosses the 900px cutoff after load (a resize, a
  tablet rotation) instead of drifting out of sync with the CSS layout underneath it.
  `.strengths-aside` deliberately carries no `.reveal` class, unlike other section intros — this pin
  is transform-type (required under ScrollSmoother), which leaves the element's own transform alone
  instead of overriding it, so `.reveal`'s CSS-transition transform would still be live and stack
  additively with the pin's, offsetting the pinned title dozens of px off from its intended spot.
  Right column `.strengths-cards` is four
  `.card` items (originally three cards + a separate `.stat-cell` stat strip, later rewritten as a
  fourth card for visual consistency; `.stat-cell`/`.strengths__stats` CSS is dead code now, not a
  reusable primitive — nothing in the markup references either class) stacked vertically, each with
  its own (non-pinned) `ScrollTrigger` in main.js that fades/tips it down into place
  (`opacity`/`y`/`rotateX`, `scrub: 1`) as it crosses into view — `.strengths-cards`' `perspective`
  gives that `rotateX` real depth. Below 900px both columns collapse into one
  (`.strengths-layout { display: block }`, JS pin torn down via `matchMedia`), same cutoff
  `.about__cards` uses.
  This replaced an earlier pinned horizontal-scroll 3D-fan carousel (and the curtain-dim transition
  into `#match` that was tied to its pin height, hence `#match` no longer needs the
  `position: relative` that curtain painting required) — nothing in this layout resizes the page, so
  it needs no extra `ScrollTrigger.refresh()` either.
- **ScrollSmoother wrapper**: `#smooth-wrapper > #smooth-content` wraps everything in `<body>`
  *except* `.skip-link`, the hidden `#grainNoise` `<svg>`, and `.grain` — those stay as plain
  siblings before the wrapper so they keep native `position: fixed` behavior (a `transform` on an
  ancestor redefines the containing block for `position: fixed`/`absolute` descendants, and
  `ScrollSmoother` transforms `#smooth-content`). A guarded first IIFE calls `ScrollSmoother.create({
  smooth: 1.4, effects: false, normalizeScroll: true })`, pins `.nav` for the rest of the page
  (`ScrollTrigger.create({ trigger: '.nav', start: 'top top', endTrigger: 'footer', end: 'bottom
  bottom', pin: true, pinSpacing: false })`), and rewrites `a[href^="#"]` clicks to
  `smoother.scrollTo(smoother.offset(target) - navHeight, true)` instead of native anchor jump.
  **Why `ScrollTrigger.pin` instead of CSS `position: sticky`** for `.nav`/`.intro`:
  `ScrollSmoother` scrolls by transforming `#smooth-content`, not by changing `#smooth-wrapper`'s
  real `scrollTop` — so `position: sticky` (computed from the scrolling ancestor's actual offset)
  would never register as scrolled and would silently stop sticking. `ScrollTrigger.pin` is wired
  directly into `ScrollSmoother`'s virtualized scroll position instead. The CSS `position: sticky`
  rules on `.nav`/`.intro` stay in the stylesheet as the fallback for whenever the first IIFE bails
  out (reduced motion, or any GSAP script failing to load) and `ScrollSmoother.create()` never
  runs. The intro's `ScrollTrigger` pin only needs `gsap`+`ScrollTrigger` (not `ScrollSmoother`), so
  a `ScrollSmoother`-only failure still leaves that sequence working — only the page-wide inertia
  scroll and nav pin fall back to `.nav`'s CSS `sticky`.
